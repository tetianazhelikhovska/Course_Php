================================================================================
АНОТАЦІЯ ПРОЕКТУ CMS
================================================================================

ЗАГАЛЬНИЙ ОПИС
Це мікро-CMS система на PHP з розділенням на адміністративну і публічну частини. 
Проект реалізований з використанням принципів MVC, IoC контейнера та репозиторіїв 
для роботи з даними. Система побудована на PDO для роботи з MySQL БД.

================================================================================
ФУНКЦІОНАЛЬНІ КОМПОНЕНТИ
================================================================================

1. МАРШРУТИЗАЦІЯ (index.php)
   Головна точка входу, яка обробляє всі запити на основі GET параметра 'route'.
   Встановлює контейнер залежностей та направляє запити до відповідних контролерів 
   (адмін-панель, публічні сторінки, логін/логаут).

2. АУТЕНТИФІКАЦІЯ (AuthService.php)
   Сервіс керування сесіями й авторизацією для адміністраторів. Перевіряє 
   облікові дані користувача, зберігає ідентифікатор в сесії з використанням 
   password_verify() для безпеки.

3. УПРАВЛІННЯ СТОРІНКАМИ - АДМІН (PagesAdminController.php)
   Контролер для CRUD операцій сторінок: перегляд списку, створення нових, 
   редагування та видалення. Валідує slug, перевіряє унікальність та 
   забезпечує перенаправлення користувача після операцій.

4. УПРАВЛІННЯ СТОРІНКАМИ - ПУБЛІКА (PagesController.php)
   Контролер для виведення сторінок на веб-сайті. Отримує сторінку за slug-ом 
   з БД через репозиторій і рендерить відповідний вью, або показує помилку 404.

5. РЕПОЗИТОРІЙ СТОРІНОК (PagesRepository.php)
   Шар роботи з БД, який містить методи для отримання сторінок за ID чи slug-ом, 
   створення, редагування, видалення та перевірки унікальності slug-а. 
   Використовує PDO з prepared statements для захисту від SQL-інєкцій.

6. МОДЕЛЬ СТОРІНКИ (PageModel.php)
   PHP клас, який представляє одну сторінку з властивостями (id, title, slug, content). 
   Використовується репозиторієм для маппінгу результатів BD запитів в об'єкти.

7. ЗАХИСТ ВІД CSRF (CsrfHelper.php)
   Утиліта для генерації та перевірки CSRF токенів. При POST запитах перевіряє 
   наявність валідного токена в $_POST, блокує запит при невідповідності 
   та регенерує токен після успішної валідації.

8. КОНТЕЙНЕР ЗАЛЕЖНОСТЕЙ (Container.php)
   IoC контейнер для управління залежностями (PDO, контролери, сервіси). 
   Дозволяє зв'язувати сервіси через функції-callback та 
   отримувати їх екземпляри при потребі.

9. ЛОГІН КОНТРОЛЕР (LoginController.php)
   Контролер для сторінки логіну. Перенаправляє залогованих користувачів 
   в адмін-панель, обробляє POST-запити для авторизації та реалізує logout.

10. ПОМИЛКА 404 (NotFoundController.php)
    Контролер для обробки маршрутів, які не знайдені. Показує сторінку 
    помилки 404 з використанням відповідного вью.

11. СЕСІЇ (session.php)
    Файл налаштування PHP сесій (вірогідно), забезпечує ініціалізацію 
    механізму сесій для зберігання даних користувача.

12. БД З'ЄДНАННЯ (db-connect.inc.php)
    Встановлює PDO з'єднання з MySQL БД (localhost, dbname=cms). 
    Перехоплює виключення PDOException і показує повідомлення про помилку.

13. СТИЛІ (styles/)
    Три CSS файли: admin.css (стилі адмін-панелі), custom.css (користувацькі), 
    simple.css (базові стилі) для оформлення інтерфейсу.

14. ШАБЛОНИ (views/)
    Розділені на адмін (views/admin/) і публіку (views/frontend/). 
    Містять HTML форми для логіну, CRUD операцій сторінок і виведення 
    контенту, а також layout шаблони та 404 сторінки помилок.

================================================================================
СТЕК ТЕХНОЛОГІЙ
================================================================================
- PHP 7.4+ (конструктори з типізацією, match/arrow functions)
- MySQL 5.7+ з підтримкою utf8mb4
- PDO для безпечної роботи з БД (prepared statements)
- Сесії PHP для зберігання стану користувача
- HTML5 & CSS3 для фронтенду

================================================================================
ПОРЯДОК РОБОТИ СИСТЕМИ
================================================================================
1. Користувач входить на сайт (index.php)
2. CsrfHelper генерує токен для форм
3. На основі маршруту контейнер створює потрібний контролер
4. Контролер запитує дані через репозиторій
5. Репозиторій запитує дані з БД через PDO
6. Результат рендериться у відповідний вью
7. Для адмін-панелі: AuthService перевіряє авторизацію перед доступом
