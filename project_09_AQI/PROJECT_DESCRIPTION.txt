Назва проєкту: AQI — Індекс якості повітря

Анотація:
Цей проєкт призначений для відображення, аналізу та візуального представлення показників якості повітря у різних містах. Він збирає структуровані дані про індекс якості повітря (AQI) і надає зрозумілий інтерфейс для перегляду поточних значень, моніторингу змін у часі та порівняння локацій. Мета проєкту — покращити обізнаність громадськості та допомогти користувачам приймати обґрунтовані рішення щодо здоров'я, планування щоденних активностей та реагування на екологічні події. Проєкт також може служити як базова платформа для подальшого розвитку аналітичних інструментів і інтеграції з зовнішніми сервісами.

Опис функцій:
Джерела даних та імпорт: Проєкт зчитує структуровані дані з локального файлу `data/index.json`, де зберігається інформація по містах, часові мітки та параметри забруднення. Архітектура дозволяє легко підключити зовнішні API (наприклад, державні або комерційні сервіси моніторингу повітря) та автоматизувати оновлення даних через cron або webhook. Парсинг і нормалізація даних відбуваються на серверній частині PHP, що гарантує коректність формату перед візуалізацією. Такий підхід забезпечує гнучкість при зміні джерел і зменшує ризик неконсистентних записів у інтерфейсі.

Відображення та представлення даних: Головна сторінка надає зручний огляд по містах з поточними значеннями AQI, кольоровими індикаторами та короткими повідомленнями про стан повітря. Сторінка конкретного міста (`city.php`) демонструє детальну інформацію — розбиття за основними параметрами (PM2.5, PM10, O3 тощо), історичні ряди та текстові пояснення значень. Інтерфейс стилізований мінімалістично за допомогою `styles/simple.css`, що забезпечує швидке завантаження, хорошу читабельність та адаптивність на різних пристроях. Користувацький інтерфейс орієнтований на простоту: інформація подається у вигляді карток, таблиць та діаграм для швидкого сприйняття.

Візуалізації та взаємодія: Для побудови інтерактивних графіків використовується `scripts/chart.umd.js`, що дозволяє відображати часові тренди, порівнювати періоди та показувати підказки при наведенні курсора. Користувач може вибирати інтервали часу, масштабувати графіки та переключатися між різними параметрами забруднення для детального аналізу. Діаграми підтримують легенди, кольорові маркери та можливість експорту зображень або даних у CSV для подальшої обробки. Це робить інструмент корисним як для звичайних користувачів, так і для аналітиків або локальних активістів з екології.

Архітектура та розширюваність: Код організовано модульно — загальні функції розміщені в `inc/functions.inc.php`, а шаблони інтерфейсу у `views/header.inc.php` та `views/footer.inc.php`. Логіка сторінок розділена між `index.php` (агрегований огляд) та `city.php` (деталізація для конкретного міста), що полегшує навігацію та підтримку коду. Така структура дозволяє легко додавати нові можливості: фільтри за діапазонами AQI, нотифікації про погіршення якості повітря, експорт/імпорт даних, або підключення бази даних для збереження великих історій вимірювань. Розробники можуть доповнювати систему API-ендпоінтами, авторизацією або інтеграціями з мобільними застосунками без значної переробки існуючого коду.

Додаткові можливості та сценарії використання: Проєкт може розширюватися до платформи для громадських ініціатив і наукових досліджень завдяки простому формату даних та відкритій структурі. Можливі сценарії включають формування щоденних звітів, відправлення сповіщень користувачам при перевищенні порогів, або побудову карт забруднення для міської адміністрації. Також передбачено місце для реалізації модулів прогнозування на основі історичних даних та машинного навчання. Загалом, платформа поєднує легкість використання з потенціалом для професійного аналітичного застосування.
